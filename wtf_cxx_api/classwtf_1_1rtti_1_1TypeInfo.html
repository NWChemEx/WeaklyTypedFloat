Detailed Description

Custom RTTI class for floating-point types.

C++'s RTTI class is a bit lacking. Ours is better :wink:. In particular, ours standardizes the state so that we know we get say the same name for the same floating-point type across different compilers. It also provides support for distinguishing between const and non-const types. Finally, it provides some introspection properties, such as the precision of the type and whether or not the type can be implicitly converted to another type.

Note
To create instances of this class, use the wtf_typeid function.

The name of the type held by *this.

This method is used to get a human-readable name for the type held by *this. The names are standardized by WTF so that they will be the same across compilers (unlike typeid(T).name(), which is implementation defined). The names will also include const qualifiers.

Can the type held by ti1 be implicitly converted to the type held by ti2?

Template Parameters
TupleTypeA std::tuple of types. TupleType must contain the type that both ti1 and ti2 hold. If it doesn't an exception will be raised. In practice, users should provide a std::tuple of all types they want to support.

This function essentially checks if a function like:

ti2 convert(ti1) { return std::declval<ti1>(); }

would compile. In this function ti1 and ti2 are the types held by ti1 and ti2 respectively.

Parameters
[in]ti1The type info for the type we want to convert from.
[in]ti2The type info for the type we want to convert to.
Returns
True if the type held by ti1 can be implicitly converted to the type held by ti2. False otherwise.
Exceptions
std::runtime_errorif TupleType does not contain the type held by either ti1 or ti2.
